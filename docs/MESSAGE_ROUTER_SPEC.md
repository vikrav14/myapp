# Unified Message Router Spec

**Purpose:** One contract for classifying inbound user text ‚Üí structured writes ‚Üí user-visible acks.  
**Depends on:** `MAURI_MEMORY_ARCHITECTURE.md`, `VIK_TRANSCRIPT_AUDIT.md`  
**Status:** Phase 0 (schema + stub). AI integration in Phase 1.

---

## 1. Problem

Today Mauri runs **two separate AI extractors** only after a long rule chain picks ‚Äúconversational fallback‚Äù:

| Extractor | When | Writes |
|-----------|------|--------|
| `extractUserMindProfile` | Know-you, `remember that` | `user_mind_facts` |
| `extractStructuredContext` | Every chat message | finance / habits / todos / emotions |

Gaps this causes (see Vik audit):

- Mid-week life changes stay in chat unless the user types `remember that`.
- Sunday report ignores snapshots; roast/hype use a different week window.
- Todo completion (‚Äúdone X‚Äù) has no path.
- Two AI calls worth of latency and inconsistency when both profile and logs appear in one message.

**Goal:** One AI pass proposes a **typed routing plan**; code validates and commits writes. Rules still win for explicit commands (`pay`, `help focus`, `remind me`).

## 3. Router output contract

Defined in `src/schemas/message-router.ts` as `MessageRouterExtraction`.

### 3.1 Intent

| Intent | Meaning | Commits |
|--------|---------|---------|
| `chat_only` | Vent, banter, question ‚Äî no structured writes | memories only |
| `structured_log` | Measurable event(s) | finance / habits / todos / emotions |
| `profile_delta` | Stable truth changed | `user_mind_facts` upsert |
| `mixed` | Both logs and profile in one message | all applicable tables |
| `command` | Reserved ‚Äî rules handle; router should not emit in chat path | ‚Äî |

### 3.2 Fields

```typescript
{
  intent: "chat_only" | "structured_log" | "profile_delta" | "mixed",
  structured?: MauriBrainDumpExtraction,   // reuse existing schema
  profile_deltas?: ProfileDelta[],         // category + fact_key + fact_value
  todo_completions?: { task_match: string }[],
  open_loop_hint?: string,                 // optional nightly snapshot input
  confidence?: "high" | "medium" | "low"
}
```

### 3.3 Profile delta categories

Same taxonomy as `user_mind_facts`: `identity`, `location`, `life_context`, `interests`, `goals`, `stressors`, `relationships`, `preferences`, `boundaries`, `user_stated`.

**Ack rule:** If `profile_deltas.length > 0` and any delta is material (not a typo fix on tone), append a one-line ack to the reply:

> Got it ‚Äî updated how I read your money pressure.

Generated by `buildProfileDeltaAck()` in `message-router.service.ts` ‚Äî not free-form AI ack (keeps voice consistent).

### 3.4 Todo completion

`todo_completions[].task_match` is fuzzy-matched against open `todo_logs` for the user (last 14 days, `is_completed = false`). Code sets `is_completed = true` ‚Äî router only proposes matches.

## 4. Commit rules (code, not AI)

1. **Validate** with Zod before any DB write.
2. **Profile deltas** ‚Üí upsert on `(user_id, category, fact_key)` with `source = "inferred"`.
3. **Structured** ‚Üí existing `persistExtraction()`.
4. **Todo completions** ‚Üí new `completeMatchedTodos()` (Phase 2).
5. **Open loop hint** ‚Üí log + pass to nightly snapshot prompt (Phase 2); no immediate user message.
6. **Low confidence** ‚Üí structured writes skipped; log shadow diff; still reply.
7. **Material profile change** on finance/relationship stressors ‚Üí consider `buildLifeThreadCandidatesFromFacts` refresh (Phase 2).

## 5. Rollout phases

| Phase | Env `MESSAGE_ROUTER_MODE` | Behaviour |
|-------|---------------------------|-----------|
| **0** | `off` (default) | Schema + stub helpers only; no runtime change |
| **1** | `shadow` | Call `routeInboundMessage` in parallel with `extractStructuredContext`; log diffs; legacy path commits |
| **2** | `commit` | Router commits structured + profile + todo completion; legacy extractor removed from chat path |
| **3** | `commit` | Merge know-you into router; deprecate standalone `extractUserMindProfile` for onboarding |

Feature flag: `MESSAGE_ROUTER_MODE` in `src/lib/env.ts`.

Shadow logging shape:

```json
{
  "userId": "...",
  "messagePreview": "Paid Rs 8k to parents...",
  "legacy": { "finance": { ... } },
  "router": { "intent": "mixed", "profile_deltas": [...] },
  "diff": ["router_only:profile_delta.stressors.family_money"]
}
```

## 6. AI prompt sketch (Phase 1)

Single Gemini call with:

- JSON schema = `messageRouterExtractionJsonSchema`
- System: Mauri routing analyst ‚Äî classify intent, extract only what user **stated**, never invent amounts
- User message + optional `existing_facts` summary (top 12 facts by category)
- Temperature low (0.2)

**Do not** generate conversational reply in same call ‚Äî keeps retry/reply logic separate.

## 9. Sunday report alignment (follow-on, not router)

Router fixes **upstream memory**. Report still needs:

- `loadLatestSnapshot()` in report prompt
- Same `week_id` (Mon 00:00 UTC ‚Äì Sun 23:59 UTC) for report + roast + hype
- Open loops from snapshot + `open_loop_follow_ups`
- Week-over-week momentum delta from `weekly_reports.summary_json`
